pyinstaller 处理 opencv-python和 mediapipe时遇到的问题

发表于2022-06-03|更新于2022-06-04
|字数总计:516|阅读时长:1分钟|阅读量:

pyinstaller简单介绍这篇文章对pyinstaller的功能、使用方法、具体参数等进行了简单介绍。以下是我在具体使用过程中遇到的一些问题和解决方法。

  1. opencv-pythonmodule不能够找到

    • 更换版本之前将原本的版本卸载pip uninstall opencv-python

    • pyinstaller适用的opencv-python的版本为4.3.0.38

    • 鉴于pip install opencv-python==4.3.0.38下载速度过慢，这里给出网页下载.whl文件的网址：opencv_python-4.3.0.38-cp38-cp38-win_amd64.whl

    • 下载完成后进入下载文件夹，pip install opencv_python-4.3.0.38-cp38-cp38-win_amd64.whl，至此安装完毕。

  2. mediapipe module不能被找到

    方法一：

    • 若使用的是-D指令，则可以在生成好dist文件夹之后，将对应缺失的库文件，放在EXE文件对应的同级目录下，之后可以成功运行EXE。

    方法二：

    • 无论是什么操作-D或是-F，遇到类似于某些库文件找不到的情况时。
    • 可以加入参数：--add-data="path_to_module"

    其他：

    • 若存在ModuleNotFoundError:No module named **缺少包，而不是缺少包里面的某一个方法或属性，在源码中import下，或者在打包的时候加上--hidden-import = **

  3. 是否能够检测所有库

    这个问题关键在于能否在命令行运行代码的主文件，如果可以运行则说明可以找到所需要的库，否则pyinstaller打包时不会发现需要的库。

    注意一个问题，如果使用了anaconda来进行环境管理，pyinstaller最好和代码文件在同一个env中，否则pyinstaller无法找到对应库的位置。

  4. 逻辑过程

    先使用-D产生dist文件夹中包含所有的内容，之后再使用-F生成对应的单个exe文件。

  5. 减少EXE文件的大小

    在函数中调用库的时候如import cv2，应该最好写成from cv2 import imshow之类的

  6. 对于有命令行的程序

    一定不要使用 -w操作，否则就不会显示出命令行。
